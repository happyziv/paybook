Paybook MVP Implementation Plan

Docker 배포 가능한 Next.js 공동가계부 MVP 구현 계획 요약.

목표와 구조

Next.js App Router, PostgreSQL, signed PIN session, React client components, Docker multi-stage build로 Paybook MVP를 만든다. PIN은 PAYBOOK_PIN 환경변수와 비교하고 DB에 저장하지 않는다.

생성할 주요 파일

  • 앱: src/app/layout.tsx, src/app/page.tsx, src/app/globals.css
  • 컴포넌트: PinGate, SetupForm, Dashboard, MobileDetailView
  • 서버 로직: config, auth, db, repository, validation, month
  • API: PIN, household, monthly, expenses add/delete
  • 배포: Dockerfile, docker-compose.example.yml, .env.example, README.md

작업 순서

  1. Next.js/TypeScript/Vitest 프로젝트 스캐폴딩
  2. 월/검증/환경변수 유틸리티와 테스트 작성
  3. PostgreSQL 스키마 자동 생성과 repository 구현
  4. 환경변수 PIN 기반 signed cookie 인증 구현
  5. 데이터 API 라우트 구현
  6. 데스크톱 대시보드와 모바일 입력 중심 화면 구현
  7. 모바일 상세내역 화면과 월 선택 흐름 구현
  8. Docker/Proxmox 배포 문서와 예제 작성
  9. unit, lint, build, Docker smoke 검증

검증 기준

  • PIN 없이는 데이터 API가 401을 반환한다.
  • 두 사람 이름 설정 후 월별 합계와 상세 내역이 보인다.
  • 쓴 사람은 선택형으로 저장된다.
  • 최근/자주 쓴 용도 5개를 선택하거나 자유 입력할 수 있다.
  • 모바일 기본 화면은 입력 중심이고, 오른쪽 위 상세내역 버튼으로 월별 내역을 확인한다.
  • 모바일 상세내역 화면에서 다른 달을 선택해 합계와 내역을 볼 수 있다.
  • Docker 이미지는 DATABASE_URL, PAYBOOK_PIN, PAYBOOK_SESSION_SECRET만으로 실행된다.
